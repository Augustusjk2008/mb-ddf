# 编译与调试指南

本文档旨在介绍如何在 Windows 环境下使用提供的批处理脚本进行项目的编译、部署与调试。

## 1. 编译项目 (build.bat)

`build.bat` 是一个封装了 PowerShell 脚本的批处理文件，用于在 Windows 上进行交叉编译。

### 1.1 基础用法

在终端（PowerShell）中运行以下命令：

*   **编译调试版本**：
    ```bash
    .\build.bat debug
    ```
*   **编译发行版本**：
    ```bash
    .\build.bat release
    ```
*   **清理工程**：
    ```bash
    .\build.bat clean
    ```

### 1.2 更改默认工程名

脚本默认的工程名称为 `DemoProject`。如果需要更改编译出的可执行文件名，请按以下步骤操作：

1.  使用文本编辑器打开项目根目录下的 `build.bat`。
2.  找到以下行：
    ```batch
    set "PROJECT_NAME_OVERRIDE=DemoProject"
    ```
3.  将 `DemoProject` 更改为您想要的名称，例如：
    ```batch
    set "PROJECT_NAME_OVERRIDE=MyNewApp"
    ```
4.  保存文件并重新运行编译命令。

### 1.3 脚本参数说明

`build.bat` 支持透传参数给底层的编译引擎，常用参数包括：

| 参数 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `-ProjectName` | 指定生成的可执行文件名 | `DemoProject` |
| `-Jobs` | 指定并行编译的线程数 | `8` |
| `-Sysroot` | 指定 aarch64 的 sysroot 路径 | 自动从工具链识别 |
| `-ToolchainBin`| 指定交叉编译工具链的 `bin` 目录 | 自动识别 |

**示例**：指定 4 线程编译并命名为 `TestApp`
```bash
.\build.bat debug -ProjectName TestApp -Jobs 4
```

## 2. 调试项目 (debug.bat)

`debug.bat` 用于将编译好的可执行文件拷贝到目标板，并启动 `gdbserver` 进行远程调试。它还会自动配置 VS Code 的调试环境。

### 2.1 基础用法

确保目标板已联网且与开发机互通，然后运行：
```bash
.\debug.bat
```

运行后，脚本会执行以下操作：
1.  自动更新 `.vscode/launch.json` 中的 `miDebuggerPath`（GDB 路径）和 `sysroot`。
2.  通过 SSH 连接目标板。
3.  同步可执行文件到目标板的指定目录。
4.  在目标板上启动 `gdbserver`。

此时，您只需在 VS Code 中按下 `F5` 即可开始调试。

### 2.2 直接运行 (Release模式)

如果您只需要在目标板上运行程序并查看输出，而不需要进行交互式调试，可以使用 `run` 参数：

```bash
.\debug.bat run
```

使用 `run` 参数后，脚本的行为会发生以下变化：
1.  **自动编译 Release 版本**：脚本会调用 `build.bat release` 重新编译项目。
2.  **上传并运行**：将 Release 版本的可执行文件同步到目标板。
3.  **用户确认**：在正式运行前，会提示 `Deploy complete. Do you want to run the program? [Y/n]`。
4.  **实时输出**：直接在当前的 Windows 终端中显示程序在目标板上运行的实时输出，而不是启动 `gdbserver`。

### 2.3 脚本参数说明

如果您的环境与默认配置不同，可以使用以下参数：

| 参数 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `-RemoteHost` | 目标板的 IP 地址 | `192.168.1.29` |
| `-RemoteUser` | SSH 登录用户名 | `root` |
| `-RemoteDir` | 目标板上的存放路径 | `/home/sast8/tmp` |
| `-RemoteGdbPort`| GDB 调试端口 | `1234` |

**示例**：连接 IP 为 `192.168.1.50` 的设备进行调试
```bash
.\debug.bat -RemoteHost 192.168.1.50
```

---

## 3. 注意事项

*   **环境依赖**：请确保系统已安装 PowerShell（Windows 7 及以上系统自带）且已配置交叉编译工具链的环境变量。
*   **权限问题**：脚本会自动处理执行策略（Bypass），无需手动修改 PowerShell 的 `ExecutionPolicy`。
*   **路径空格**：建议项目路径中不要包含中文或空格，以免编译工具链报错。
