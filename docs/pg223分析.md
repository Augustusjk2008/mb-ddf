# Xilinx CAN FD v2.0 寄存器配置流程（1M/1M波特率+回环模式+CAN FD标准帧收发）
基于补充的完整文档，进一步明确**CAN FD普通帧**的核心定义：非扩展帧（11位标准ID）、非远程帧（无RTR位）、CAN FD格式（FDF=1），且仲裁/数据相位波特率均为1Mbit/s，回环模式下自收自发。以下是结合文档细节优化后的完整配置流程，确保寄存器操作与手册完全一致。

## 核心前提（文档明确要求）
1. 配置仅在**Configuration模式**（`SRR[CEN]=0`）下有效，修改后需置`CEN=1`退出该模式；
2. 软件复位（`SRR[SRST]=1`）后必须等待**16个AXI4-Lite/APB时钟周期**，避免寄存器读写冲突；
3. CAN时钟推荐采用Bosch建议的**20MHz**（文档第77页），波特率计算基于此时钟；
4. 所有定时类寄存器（波特率预分频、位定时段）的**实际值=写入值+1**（文档第17、18页）；
5. CAN FD帧无远程帧格式，`TB-ID[SRR/RTR/RRS]`和`TB-ID[RTR/RRS]`必须置0（文档第45页）。

## 步骤1：软件复位与配置模式进入
**目标**：清空核心配置，确保核心处于可配置状态，寄存器**SRR（0x0000）-软件复位寄存器**（文档第14页）。
1. 写`SRR = 0x00000001`：置`SRST=1`触发软复位，所有配置寄存器恢复默认，复位后该位自动清0；
2. 等待**16个AXI4-Lite/APB时钟周期**（文档第14页Note）；
3. 写`SRR = 0x00000000`：保持`CEN=0`，核心进入Configuration模式（`SR[CONFIG]=1`），此时可修改所有核心配置寄存器。

## 步骤2：配置回环模式（禁止其他模式冲突）
**目标**：使能回环模式，禁止睡眠、侦听模式，寄存器**MSR（0x0004）-模式选择寄存器**（文档第15-16页）。
写`MSR = 0x00000002`，关键位严格按文档配置：
| 位域 | 名称 | 写入值 | 文档依据 |
|------|------|--------|----------|
| 7    | ABR  | 0      | 关闭自动Bus-Off恢复，默认配置 |
| 6    | SBR  | 0      | 关闭手动Bus-Off恢复，默认配置 |
| 5    | DPEE | 0      | 使能协议异常检测（CAN FD帧需此配置） |
| 4    | DAR  | 0      | 使能自动重传（CAN FD帧默认特性） |
| 3    | BRSD | 0      | 不覆盖BRS位，按TB-DLC的BRS配置执行 |
| 2    | SNOOP| 0      | 关闭侦听模式（回环模式下不可同时使能） |
| 1    | LBACK| 1      | 使能回环模式（核心自收自发） |
| 0    | SLEEP| 0      | 关闭睡眠模式（回环模式下不可同时使能） |
**关键约束**：`LBACK/SLEEP/SNOOP`不可同时置1（文档第15页IMPORTANT），否则模式配置无效。

## 步骤3：1M/1M波特率配置（仲裁/数据相位一致）
波特率公式：`波特率 = CAN_CLK / [(BRP+1) × (TS1+1 + TS2+1)]`（文档第17页），20MHz时钟下配置1M波特率需满足`(BRP+1)×(TS1+1+TS2+1)=20`，选择`BRP=1、TS1=6、TS2=2`（实际值：2×(7+3)=20），仲裁/数据相位配置完全一致。

### 3.1 仲裁相位（Nominal）波特率配置
1. **BRPR（0x0008）-仲裁相位预分频寄存器**（文档第17页）：写`0x00000001`，`BRP[7:0]=1`，实际预分频=2；
2. **BTR（0x000C）-仲裁相位位定时寄存器**（文档第18页）：写`0x00002601`，关键位：
   - `SJW[22:16]=2`：实际同步跳转宽度=3（与TS2一致，文档推荐配置）；
   - `TS2[14:8]=2`：实际相位段2=3；
   - `TS1[7:0]=6`：实际相位段1=7（传播段+相位段1）。

### 3.2 数据相位（Data）波特率配置
CAN FD帧无波特率切换（`BRS=0`），数据相位配置与仲裁相位一致，同时禁用发送延迟补偿（TDC，仅FD高速模式需启用）：
1. **DP_BRPR（0x0088）-数据相位预分频寄存器**（文档第29页）：写`0x00000001`，关键位：
   - `TDC[16]=0`：禁用发送延迟补偿（普通CAN FD帧无需）；
   - `TDCOFF[13:8]=0`：延迟补偿偏移=0；
   - `DP_BRP[7:0]=1`：实际预分频=2；
2. **DP_BTR（0x008C）-数据相位位定时寄存器**（文档第30页）：写`0x00000621`，关键位：
   - `DP_SJW[19:16]=2`：实际同步跳转宽度=3；
   - `DP_TS2[11:8]=2`：实际相位段2=3；
   - `DP_TS1[4:0]=6`：实际相位段1=7。

## 步骤4：使能收发中断（可选，简化状态判断）
**目标**：通过中断快速判断收发成功，寄存器**IER（0x0020）-中断使能寄存器**（文档第25-26页）。
写`IER = 0x00000006`，关键位：
- `ETXOK[1]=1`：使能发送成功中断（TXOK）；
- `ERXOK[4]=1`：使能接收成功中断（RXOK）；
**中断清除**：后续通过`ICR`寄存器清除标志，核心中断输出为`ip2bus_intrevent`（高有效，文档第9页）。

## 步骤5：使能核心，退出配置模式
**目标**：启动核心，进入回环模式，寄存器**SRR（0x0000）**（文档第14页）。
1. 写`SRR = 0x00000002`：置`CEN=1`，核心开始初始化；
2. 等待核心检测**11个连续隐性位**（回环模式下内部自动生成，文档第65页）；
3. 确认模式：读**SR（0x0018）-状态寄存器**（文档第20-21页），验证`SR[LBACK]=1`、`SR[CONFIG]=0`、`SR[NORMAL]=0`，表示核心成功进入回环模式。

## 步骤6：发送CAN FD标准帧（非扩展、非远程）
**目标**：配置TX缓冲区，发送标准ID（0x123）、4字节数据的CAN FD帧，基于**TB0缓冲区（0x0100起始）**（文档第42-46页）。

### 6.1 配置TB0-ID寄存器（标准ID，非扩展）
**TB0-ID（0x0100）-TX0 ID寄存器**（文档第45页）：写`0x00000123`，关键位严格按CAN FD标准帧要求：
| 位域          | 名称          | 写入值 | 文档依据 |
|---------------|---------------|--------|----------|
| 31:21         | ID[28:18]     | 0x123  | 11位标准ID（非扩展帧，IDE=0） |
| 20            | SRR/RTR/RRS   | 0      | 非远程帧，CAN FD帧无远程模式 |
| 19            | IDE           | 0      | 非扩展帧，禁用扩展ID |
| 18:1          | ID[17:0]      | 0      | 非扩展帧，此位域无效，置0 |
| 0             | RTR/RRS       | 0      | 非远程帧，CAN FD帧无远程模式 |

### 6.2 配置TB0-DLC寄存器（CAN FD格式，无波特率切换）
**TB0-DLC（0x0104）-TX0数据长度寄存器**（文档第46页）：写`0x44000000`，关键位：
| 位域          | 名称          | 写入值 | 文档依据 |
|---------------|---------------|--------|----------|
| 31:28         | DLC[3:0]      | 4      | 数据长度4字节（CAN FD支持0-64字节） |
| 27            | FDF（原EDL）  | 1      | CAN FD格式帧核心标志（区分CAN帧） |
| 26            | BRS           | 0      | 不切换波特率（数据相位=仲裁相位=1M） |
| 25            | Reserved      | 0      | 保留位，强制置0 |
| 24            | EFC           | 0      | 关闭TX事件FIFO存储（无需记录发送事件） |
| 23:0          | MM/Reserved   | 0      | 消息标记/保留位，置0 |

### 6.3 写入发送数据
**TB0-DW0（0x0108）-TX0数据寄存器0**（文档第46页）：写`0x11223344`，对应4字节数据：
- 31:24 = 0x11（Data Byte 0）；
- 23:16 = 0x22（Data Byte 1）；
- 15:8 = 0x33（Data Byte 2）；
- 7:0 = 0x44（Data Byte 3）；
**注意**：若DLC>4，需继续写TB0-DW1~DW15（每寄存器存储4字节），仅需写DLC对应长度的寄存器。

### 6.4 触发发送与状态确认
1. 写**TRR（0x0090）-TX缓冲区就绪请求寄存器**（文档第31页）：`TRR = 0x00000001`，置`RR0=1`，触发TB0帧发送；
2. 确认发送成功：
   - 轮询`ISR（0x001C）-中断状态寄存器`（文档第22-24页），`ISR[TXOK]=1`表示发送成功；
   - 核心发送完成后自动清`TRR[RR0]=0`，可辅助验证；
3. 清除发送标志：写**ICR（0x0024）-中断清除寄存器**（文档第27-28页）：`ICR = 0x00000002`，置`CTXOK=1`，清`ISR[TXOK]`。

## 步骤7：接收CAN FD标准帧（回环模式自收）
回环模式下，核心发送的CAN FD帧自动回环至RX端，经ID过滤后存入**RX FIFO0（0x2100起始，64深度）**（文档第50-51页），接收流程如下：

### 7.1 检查接收状态
- 轮询`FSR（0x00E8）-RX FIFO状态寄存器`（文档第39-40页）：`FSR[FL[14:8]]>0`表示FIFO0有数据；
- 轮询`ISR（0x001C）`：`ISR[RXOK]=1`表示接收成功（回环模式下RXOK先于TXOK置1，文档第24页Note）。

### 7.2 读取接收数据
从RX FIFO0起始地址0x2100读取，与发送帧格式一致（文档第53-55页）：
1. **RB0-ID（0x2100）-RX0 ID寄存器**：读取值=0x00000123（标准ID，IDE=0）；
2. **RB0-DLC（0x2104）-RX0数据长度寄存器**：读取值=0x44000000（FDF=1、BRS=0、DLC=4）；
3. **RB0-DW0（0x2108）-RX0数据寄存器0**：读取值=0x11223344（与发送数据一致）；
**注意**：RX寄存器为只读，禁止主机写入（文档第50页IMPORTANT）。

### 7.3 更新FIFO读指针与清除标志
1. 写`FSR（0x00E8）`：置`FSR[IRI[7]]=1`（文档第39页），核心自动将FIFO0读指针+1，填充级别`FL`自动减1；
2. 清除接收标志：写`ICR（0x0024）= 0x00000010`（文档第28页），置`CRXOK=1`，清`ISR[RXOK]`。

## 关键验证点（确保CAN FD帧特性）
1. 发送端：`TB0-DLC[FDF]=1`（0x0104寄存器第27位为1），确认是CAN FD帧；
2. 接收端：`RB0-DLC[FDF]=1`（0x2104寄存器第27位为1），确认接收的是CAN FD帧；
3. 非扩展帧：`TB0-ID[IDE]=0`、`RB0-ID[IDE]=0`，仅11位标准ID有效；
4. 非远程帧：`TB0-ID[RTR/RRS]=0`、`RB0-ID[RTR/RRS]=0`，符合CAN FD帧规范；
5. 波特率：仲裁/数据相位均为1Mbit/s，无切换（`BRS=0`）。

## 文档关键约束与注意事项
1. 配置模式下修改寄存器后，需等待11个连续隐性位才能退出（文档第65页），否则核心无法正常工作；
2. TX/RX缓冲区为块RAM，软/硬复位不会清空（文档第42页Note），需主机手动管理；
3. 若需发送64字节数据，需写TB0-DW0~DW15（16个寄存器×4字节=64字节），DLC字段需设为0x0F（CAN FD DLC=0x0F对应64字节，文档第46页）；
4. 所有CAN FD节点需保持波特率、采样点一致（文档第89页），否则会触发帧错误（ESR寄存器相关位会置1）。
